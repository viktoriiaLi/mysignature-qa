# Evidence — CASE-001  
**Google Login → Onboarding Bypass**

---

## Video Evidence 1  
**Scenario:** Initial login via Google with onboarding bypass

**Link:**  
https://drive.google.com/file/d/1t-mcoZZXJ7bQvfkn1IGxdBF8jFY10yN/view?usp=share_link

**Description:**  
User logs in via Google, skips mandatory onboarding step (**Position** field),  
refreshes the page and gains access to the dashboard.

---

## Video Evidence 2  
**Scenario:** Re-login after partial onboarding

**Link:**  
https://drive.google.com/file/d/1suC0Yr6J2qVooaJrL4w_LAGtcQyURRSG/view?usp=share_link

**Description:**  
User logs out, logs in again via Google,  
is prompted to enter name, refreshes the page,  
and is redirected to dashboard with previously saved name —  
onboarding is not enforced.

---

## Summary
Mandatory onboarding step (**Position**) can be bypassed  
via page refresh or re-login flow when using Google authentication.
